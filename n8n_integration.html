n8n Integration Guide

Complete setup guide for connecting your n8n workflows to the Predictive Maintenance API

API Endpoint Seçimi

Normal API (Tüm Parametreler)

Endpoint: /predict

Kullanım: Çalışma saatleri dahil tüm parametreler

Avantaj: En doğru tahmin sonuçları

Dezavantaj: Çalışma saatleri verisi gerekli

Çalışma Saatleri Olmadan

Endpoint: /predict_without_hours

Kullanım: Çalışma saatleri olmadan tahmin

Avantaj: Çalışma saatleri verisi gerektirmez

Dezavantaj: Sabit 25,000 saat varsayımı

n8n Hızlı Kurulum

HTTP Request Node Ayarları:
  • Method: POST
  • URL (Normal): {{ request.url_root }}predict
  • URL (Çalışma Saatleri Olmadan): {{ request.url_root }}predict_without_hours
  • Send Body: ✅ Enabled
  • Content-Type: application/json
  • Authentication: None required
Response Format:
  • Status: 200 OK
  • Type: JSON Object
  • Fields: 3 prediction values
  • Error Handling: 500 on error

Hangi API'yi Kullanmalıyım?

Normal API (/predict) Kullan
  • Çalışma saatleri verisi mevcut
  • En yüksek doğruluk gerekli
  • Prodüksiyon ortamı
  • Kritik kararlar alınacak
  • Makine çalışma saatleri güvenilir
Çalışma Saatleri Olmadan API (/predict_without_hours) Kullan
  • Çalışma saatleri verisi yok
  • Test/geliştirme ortamı
  • Hızlı tahmiller gerekli
  • Sensör verilerine odaklan
  • Operasyonel saatler güvenilmez
Öneri: Çalışma saatleri veriniz varsa normal API'yi kullanın. Yoksa veya güvenilir değilse çalışma saatleri olmadan API'yi tercih edin.

n8n için Dinamik Veri Eşleme

Use these expressions in your n8n workflow to map dynamic data:

Field n8n Expression Example Description
Temperature_C {{ "{{" }} $json.sensor_data.temperature {{ "}}" }} Map from sensor temperature reading
Vibration_mms {{ "{{" }} $json.vibration_sensor {{ "}}" }} Map from vibration sensor data
Power_Consumption_kW {{ "{{" }} $json.power_meter.current_kw {{ "}}" }} Map from power consumption meter
Operational_Hours {{ "{{" }} $json.machine.total_hours {{ "}}" }} Map from machine runtime counter (Normal API için)
Machine_Type {{ "{{" }} $json.machine.type || "Type_A" {{ "}}" }} Map machine type with fallback

Response Handling in n8n

Success Response (200):
{
  "Failure_Within_7_Days": false,
  "Failure_Probability": 0.2345,
  "Remaining_Useful_Life_days": 45.67
}
Error Response (500):
{
  "error": "Detailed error message"
}
Access Response Data in n8n:
  • {{ "{{" }} $json.Failure_Within_7_Days {{ "}}" }} - Boolean: true/false
  • {{ "{{" }} $json.Failure_Probability {{ "}}" }} - Float: 0.0 to 1.0
  • {{ "{{" }} $json.Remaining_Useful_Life_days {{ "}}" }} - Float: days remaining

n8n Workflow Examples

Workflow: Schedule → Get Sensor Data → Predict → Send Alert

  1. Cron Node: Schedule every hour
  2. HTTP Request: Get machine sensor data
  3. HTTP Request: Call prediction API
  4. IF Node: Check if Failure_Within_7_Days == true
  5. Email/Slack: Send alert if high risk

Workflow: Multiple Conditions + Database Logging

  1. Webhook: Receive sensor data
  2. HTTP Request: Call prediction API
  3. Switch Node: Route based on probability levels:
    • High Risk (>70%): Immediate alert
    • Medium Risk (40-70%): Warning notification
    • Low Risk (<40%): Log only
  4. Database: Log all predictions
  5. Multiple Outputs: Email, Slack, SMS based on severity

Workflow: Real-time IoT Data Processing

  1. MQTT Trigger: Listen to IoT sensor data
  2. Function Node: Transform and validate sensor data
  3. HTTP Request: Call prediction API with formatted data
  4. Switch Node: Handle different response scenarios
  5. InfluxDB/TimescaleDB: Store time-series prediction data
  6. Grafana Webhook: Update dashboard alerts

Error Handling in n8n

Recommended Error Handling Strategy:
  1. Set Error Workflow: Configure n8n to handle HTTP errors gracefully
  2. Check Status Code: Use IF node to check response status
  3. Retry Logic: Add retry mechanism for temporary failures
  4. Fallback Actions: Define alternative actions when API is unavailable
Common Error Scenarios:
  • 400 Bad Request: Invalid input data format
  • 500 Internal Error: Model processing error
  • Network Timeout: API endpoint unavailable
